Контейнеризуем только PostgreSQL для локальной разработки веб-приложения (веб запускается на хосте через npm run dev), плюс отдельный one-shot сервис для Drizzle-миграций.

## Создать файлы

### 1. `docker-compose.yml`
- **db**: `postgres:17-alpine`, named volume `pgdata` для персистентности, healthcheck через `pg_isready`, порт 5432:5432. Переменные `POSTGRES_USER/PASSWORD/DB` со значениями по умолчанию `${VAR:-panda}` — compose стартует даже без `.env`.
- **migrate**: one-shot сервис на `node:22-alpine`. Монтирует репозиторий (`./:/app`) + named volume `migrate_node_modules` (Linux-бинарники, кэш установки). Команда: `npm install && npm run migrate -w @panda-lavanda/db`. В `environment` инжектит `DATABASE_URL` с хостом `db` (внутрисетевое имя). `depends_on: db (service_healthy)`, `restart: "no"`.

### 2. `.env.example`
Документирует `POSTGRES_USER/PASSWORD/DB` и `DATABASE_URL` (значение для хоста = localhost), с комментарием про подмену хоста на `db` в compose.

### 3. `.dockerignore`
node_modules, dist, .output, .nitro, .tanstack, .vinxi, .git, .env, *.log, .vscode, .idea.

### 4. `DOCKER.md` (подробная инструкция на русском)
Разделы: необходимое ПО; быстрый старт (4 команды); что где работает (БД в Docker, веб на хосте); переменные окружения (localhost vs db); работа с миграциями (generate → up → migrate); подключение к БД (psql, Drizzle Studio); полезные команды (up/down/down -v/logs/run migrate); troubleshooting (порт 5432 занят, нет миграций, пароли); короткая заметка «как позже контейнеризовать веб» (не реализуется сейчас).

### 5. `README.md`
Добавить ссылку на `DOCKER.md` после блока Getting Started.

## Порядок запуска, который зафиксирую в DOCKER.md
1. `cp .env.example .env` (опционально — есть defaults)
2. `npm install` + `npm run generate -w @panda-lavanda/db` — сгенерировать миграции (currently их нет)
3. `docker compose up -d` — поднимает БД, затем one-shot применяет миграции
4. `npm run dev` — веб локально, коннектится к localhost:5432
