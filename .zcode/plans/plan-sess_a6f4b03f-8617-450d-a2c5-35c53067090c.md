# Favorites feature — implementation plan

A LocalStorage-backed favorites system following the existing Clean Architecture layering. The user's three open questions are resolved: **(1)** `isFavoriteProduct` is a pure domain function; **(2)** `ProductCard` is extracted; **(3)** the client composition root is a new `composition-root/index.client.ts` sibling (TanStack `.client.ts` import-protection keeps the server `index.ts` graph out of the browser bundle).

---

## Layer 1 — `packages/domain`

### 1.1 New `users/` feature module

**`packages/domain/src/users/user.ts`** — entity + pure helpers:
```ts
import type { UniqueId } from '@panda-lavanda/shared'

/**
 * Authenticated or anonymous user. Until real auth exists, the web app
 * owns exactly one anonymous `IUser` whose `favoriteProductIds` are
 * persisted in browser LocalStorage (see `LocalStorageUserRepository`).
 */
export interface IUser {
  id: UniqueId
  favoriteProductIds: UniqueId[]
}

/** Whether a product is in the user's favorites. Pure, no I/O. */
export function isFavoriteProduct(user: IUser, productId: UniqueId): boolean {
  return user.favoriteProductIds.includes(productId)
}
```
No `const enum`, plain `interface` + plain `UniqueId` alias — matches `product.ts` conventions exactly.

**`packages/domain/src/users/index.ts`** — barrel:
```ts
export * from './user'
```

### 1.2 New port

**`packages/domain/src/ports/user-repository.port.ts`**:
```ts
import type { UniqueId } from '@panda-lavanda/shared'
import type { IUser } from '../users'

/**
 * Read/write the current user. Concrete implementations live in
 * infrastructure and are injected at the (server or client) composition
 * root. The current app has a single LocalStorage-backed implementation
 * for anonymous favorites; a future authenticated implementation will
 * replace it without touching callers.
 */
export interface IUserRepository {
  getCurrent(): Promise<IUser>
  toggleFavoriteProduct(productId: UniqueId): Promise<IUser>
}
```
Re-export from `packages/domain/src/index.ts`:
```ts
export * from './users'
export * from './ports/user-repository.port'
```

## Layer 3 — `packages/infrastructure`

**`packages/infrastructure/src/storage/local-storage-user.repository.ts`** — class `LocalStorageUserRepository implements IUserRepository`. Mirrors the `LocalFileStorageService` config-in-constructor pattern:
```ts
constructor(private readonly config: { storageKey: string; userId: UniqueId })
```
- Reads/writes `localStorage[storageKey]` (one entry per user, JSON `{ id, favoriteProductIds }`).
- `getCurrent()`: parse entry; if missing/corrupt, seed an empty user `{ id: config.userId, favoriteProductIds: [] }` and persist it.
- `toggleFavoriteProduct(productId)`: read, `[].includes() ? filter : [...ids, id]`, write, return updated `IUser`.
- All I/O wrapped so the repository remains the only place that touches the Web Storage API.

`storage/` is the documented home for "localStorage / DB adapters" (`architecture.md:28`).

**Update** `packages/infrastructure/src/index.ts`: add `export * from './storage/local-storage-user.repository'`.

⚠️ **TS gotcha** (`AGENTS.md`): infrastructure's tsconfig sets `"types": ["node"]`. The DOM `localStorage` global comes from the root tsconfig `"lib": ["DOM", "DOM.Iterable"]`, which is still active — no config change needed, but I'll confirm with `tsc --noEmit` after.

## Layer 4 — `apps/web`

### 4.1 Add TanStack Query

```
npm install @tanstack/react-query
```
The existing `@tanstack/react-router-ssr-query` dep is for SSR dehydration, but for a client-only favorites page we only need the base `QueryClientProvider`. Wire it in **`apps/web/src/app/routes/__root.tsx`**: add a module-level `const queryClient = new QueryClient()` and wrap `{children}` in `<QueryClientProvider client={queryClient}>` inside `RootDocument`. (No SSR dehydration needed since the favorites page is `ssr: false`.)

### 4.2 Client composition root

**`apps/web/src/app/composition-root/index.client.ts`** — new sibling:
```ts
import { LocalStorageUserRepository } from '@panda-lavanda/infrastructure'
import type { IUserRepository } from '@panda-lavanda/domain'

/**
 * Client-only composition root.
 *
 * Instantiates client-side infrastructure (browser LocalStorage). The
 * `.client.ts` suffix makes TanStack Start reject any import of this file
 * from a server bundle, and — critically — keeps it out of the server
 * `index.ts` graph so postgres-js / Node `fs` never reach the browser.
 */
export const userRepository: IUserRepository = new LocalStorageUserRepository({
  storageKey: 'panda-lavanda:user',
  userId: 'anonymous', // placeholder until real auth exists
})
```
The sibling file + suffix guarantees the server `index.ts` import graph (with its module-scope `createDb()` side effect) never reaches the browser bundle.

### 4.3 `useFavorites()` hook

**`apps/web/src/shared/hooks/use-favorites.ts`** — wraps the use cases in TanStack Query:
- `useQuery({ queryKey: ['user'], queryFn: () => new GetCurrentUserUseCase(userRepository).execute() })` for the cached `IUser`.
- `useMutation` calling `new ToggleFavoriteProductUseCase(userRepository).execute(productId)`, with `onSettled: () => queryClient.invalidateQueries({ queryKey: ['user'] })` so every heart icon on the page re-renders automatically after a toggle (this is the optimistic-rendering benefit the user asked about).
- Exposes `{ isFavorite(id), toggle(id), user, isLoading }`.

**`apps/web/src/shared/hooks/index.ts`** — new barrel.

### 4.4 Extract `ProductCard`

**`apps/web/src/shared/components/product-card.tsx`** — move the inline `<li>` from `catalog-page.tsx:99-141` here. New props:
```ts
interface ProductCardProps {
  product: IProduct
  isFavorite: boolean
  onToggleFavorite: () => void
}
```
Heart toggle: `<Button variant="ghost" size="icon" aria-label="В избранное" aria-pressed={isFavorite} onClick={onToggleFavorite}>` containing `<Heart className={cn(isFavorite && 'fill-current text-destructive')} />`. The Button already handles icon sizing. The `minPrice`/`isInStock`/`formatPrice` helpers move into this file (they're page-specific today but logically belong with the card).

**Update `catalog-page.tsx`**: replace the inline `<li>` with `<ProductCard ...>`. Catalog wires favorites in: `const { isFavorite, toggle } = useFavorites()` at the top of `CatalogPage`, passing per-card. Note the catalog's data flow stays SSR-loaded; only the heart toggle is client-interactive (the icon renders `isFavorite` from the cached user, which starts empty on first SSR paint and hydrates — acceptable UX for a non-indexed, non-critical affordance).

### 4.5 Favorites page + route (SPA-style, not indexed)

**`apps/web/src/app/routes/favorites.tsx`** — first `ssr: false` route in the repo:
```ts
export const Route = createFileRoute('/favorites')({
  component: FavoritesPage,
  // No loader — fully client-rendered; not indexed by search engines.
  ssr: false,
  head: () => ({
    meta: [{ name: 'robots', content: 'noindex, nofollow' }],
  }),
})
```
The `noindex` meta + `ssr: false` together guarantee the page is invisible to crawlers (no server HTML produced, explicit robots directive as belt-and-suspenders).

**`apps/web/src/presentation/pages/favorites-page/{favorites-page.tsx,index.ts}`** — mirrors `catalog-page/` folder convention:
- Reads `useFavorites()` to get `user.favoriteProductIds`.
- Calls the existing `getProducts` server function with `{ ids: favoriteProductIds }` (the `ids?: UniqueId[]` filter already exists on `IProductFilters`, so **no new server function needed** — reuse `getProducts` via a small client fetch; I'll add an `ids` field to the `getProductsInputSchema` zod schema to accept it). Renders the same `<ProductCard>` grid with hearts pre-filled.
- Empty state: "В избранном пусто" with a link to `/catalog`.

## Notes / assumptions
- Favorites use a single anonymous user (`id: 'anonymous'`) stored under LocalStorage key `panda-lavanda:user`. When real auth lands, swap `LocalStorageUserRepository` → server-backed impl in `index.client.ts` (or move to `index.ts`); the domain/application/web layers above don't change.
- The catalog page remains SSR; only the heart icon's filled state is client-driven. First paint may show an empty heart briefly until the client hydrates and reads LocalStorage — acceptable for a non-critical UI affordance.
- Header link to `/favorites` is intentionally out of scope (no header component exists yet); the route is reachable at `/favorites` directly.
